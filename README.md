# 《谁是卧底》联机版 — 设计方案

## 一、游戏规则

### 基础设定
- 4-10 人一局
- 每局抽一个词牌：大部分玩家拿 **平民词**，1-2 人拿 **卧底词**（两个词相近但不同）
- 可选模式：**白板模式** — 1 名玩家拿到空白词（什么都没有），靠听别人的描述编造
- 玩家轮流用一句话描述自己的词，**不能直接说出那个词**
- 所有人描述完后投票，票数最多的人出局

### 角色说明

| 角色 | 数量 | 特征 |
|------|------|------|
| 👤 平民 | 多数 | 拿到平民词，目标是找出卧底 |
| 🕵️ 卧底 | 1-2 人 | 拿到相近但不同的词，要隐藏自己 |
| ⬜ 白板 | 0-1 人 | 没有词，全靠演，要骗过所有人 |

### 完整胜负判定

| 条件 | 结果 | 胜者 |
|------|------|------|
| 卧底被投票出局 | 平民胜利 🎉 | 平民 |
| 白板被投票出局（白板模式） | 平民胜利 🎉 | 平民 |
| 最后存活 3 人且仍有卧底/白板 | 卧底/白板胜利 🕵️ | 卧底或白板 |
| 卧底在猜词阶段**猜对平民词** | 卧底特殊胜利 👑 | 卧底（完美伪装） |
| 白板活到最后且**猜对平民词** | 白板特殊胜利 👑 | 白板（最强演技） |
| 白板活到最后（未猜词）| 白板胜利 🎭 | 白板 |

> 权重：特殊胜利 > 普通胜利。如果卧底被投出但正确猜了平民词，算平民赢（投出优先）。

## 二、卧底 / 白板 猜词机制

### 猜词触发时机

每轮投票结束后、公布结果之前，进入 **猜词阶段**：

```
描述结束 → 投票 → 有人被投出 → 猜词阶段 → 公布结果 → 下一轮
```

- 仅 **未被投出的卧底和白板** 有资格猜词（每局游戏**仅一次**机会）
- 弹出输入框，输入你猜的平民词
- **猜对** → 触发特殊胜利，直接结算本局
- **猜错** → 无事发生，游戏继续
- 放弃猜词 → 保持沉默，游戏继续

### 猜词的策略意义

| 角色 | 猜词的动机 | 风险 |
|------|-----------|------|
| 卧底 | 平民快赢了时放手一搏 | 猜错会暴露身份（投票时更明显） |
| 白板 | 想获取特殊胜利 | 猜错会暴露自己 |
| 平民 | — | — |

### 猜词阶段的 UI

- 有资格的玩家弹出 **模态框**："你认为平民的词是什么？"
- 输入框 + 提交按钮 + 放弃按钮
- 倒计时 30 秒
- **其他玩家看到**："卧底正在思考..." 的等待提示，看不到输入内容
- 猜完后结果在整个结算阶段公布

## 三、胜利特效区分

### 平民胜利 🎉（金色 / 暖色系）

```
背景：金色粒子上升 + 暖橙色光晕
文字："🎉 平民获胜！"（大字、金色描边）
动画：彩带从两侧下落
音效：胜利号角（欢快明亮）
```

### 卧底胜利 🕵️（紫色 / 暗色系）

```
背景：紫色烟雾弥漫 + 暗光闪烁
文字："🕵️ 卧底获胜！"（紫色渐变、阴影闪烁）
动画：反派剪影在烟雾中隐现
音效：低沉悬疑 + 揭晓高潮
```

### 白板胜利 🎭（蓝绿色 / 神秘系）

```
背景：蓝绿色迷雾 + 面具图案旋转
文字："🎭 白板成功潜伏！"（蓝绿渐变、闪烁）
动画：面具揭开、真容显露效果
音效：神秘诡谲的揭晓音
```

### 卧底特殊胜利 👑（最华丽 / 暗金 + 紫红）

```
背景：暗金色火光冲天 + 紫色闪电
文字："👑 卧底完美伪装！"（暗金火焰字、剧烈闪烁）
动画：卧底剪影站在王座上，全场震动
音效：气势恢宏的史诗级 BGM
```

### 白板特殊胜利 👑（最华丽 / 炫彩 + 极光）

```
背景：彩色极光流动 + 星光粒子
文字："👑 白板影帝/影后！"（炫彩流光字）
动画：聚光灯打在白板身上，周围人震惊倒地
音效：戏剧性的 wow 音效 + 掌声
```

### 特效实现方式

- 纯 CSS 动画（keyframes + canvas 粒子）
- 不同胜利类型对应不同的 CSS class
- 结算页切换动画，先揭晓身份 → 再播放胜利特效
- 手机端做性能降级（减少粒子数量）

## 四、功能设计

### 页面 / 界面（单页应用，按阶段切换）

| 阶段 | 界面 | 功能 |
|------|------|------|
| 🏠 **首页** | 昵称输入 + 创建/加入房间 | 输入昵称，创建房间或输入6位房间号加入 |
| 🛋 **大厅** | 玩家列表 + 设置 + 开始按钮 | 显示已加入玩家，房主可调整参数并开始 |
| 🃏 **看词** | 你的词（仅自己可见） | 显示你的词语 + 角色提示（平民/卧底/白板） |
| 🎤 **描述** | 输入框 + 已描述列表 + 轮次指示 | 轮到你了输入描述，看别人的描述 |
| 🗳 **投票** | 玩家列表可点选 + 倒计时 | 点击你认为最可疑的人 |
| 🤔 **猜词** | 模态输入框（仅卧底/白板可见） | 输入猜测的平民词，或放弃 |
| 📊 **结果** | 投票详情 + 身份翻牌动画 | 显示谁被投出、TA的身份、两个词语 |
| 🏆 **结算** | 胜利特效全屏播放 + 赢家展示 | 完整的胜利动画，再来一局按钮 |

### 核心交互

- 实时聊天式描述展示，类似弹幕效果
- 投票倒计时（每人30秒描述，投票60秒）
- 看词时翻盖动画，防止偷看（其他人看屏幕也没用）
- 轮到谁了 — 高亮当前描述者，自动滚动
- 猜词阶段仅卧底/白板看到输入框，其他人看到等待提示
- 结算页分步揭晓：投票结果 → 身份亮牌 → 猜词结果 → 胜利特效
- 手机端适配：大按钮、横向滚动条、触摸友好

### 房间机制

- 6位房间号，进入即加入
- 房主可开始游戏、调整设置、踢人
- 断线重连（30秒内可重连回到房间）
- 闲置超时自动移出
- 房间设置参数：
  - 卧底人数：1 / 2
  - 白板模式：开 / 关
  - 描述时间：20 / 30 / 60 秒
  - 投票时间：30 / 60 / 90 秒
  - 是否开启猜词模式：开 / 关
  - 轮数限制：3 / 5 / 不限

## 五、数据模型

```javascript
/* ========== 房间 ========== */
Room {
  code: String(6),            // 6位房间号
  hostId: String,             // 房主 ID
  players: [Player],          // 玩家列表
  state: String,              // waiting | playing | ended
  settings: {
    spyCount: 1 | 2,          // 卧底数
    whiteMode: Boolean,        // 是否启用白板模式
    guessEnabled: Boolean,     // 是否启用猜词
    describeTime: 30,          // 每人描述时间(秒)
    voteTime: 60,              // 投票时间(秒)
    roundLimit: 3,             // 轮数限制
  },
  createdAt: Timestamp,
}

/* ========== 玩家 ========== */
Player {
  id: String,
  name: String,
  isAlive: Boolean,
  role: 'civilian' | 'spy' | 'white',
  word: String | null,          // 自己的词（白板为null）
  hasGuessed: Boolean,          // 本轮是否已使用猜词机会
  isHost: Boolean,
  isConnected: Boolean,
  joinedAt: Timestamp,
}

/* ========== 游戏（每局） ========== */
Game {
  roomCode: String,
  round: Number,                 // 当前第几轮
  phase: String,                 // describe | vote | guess | result | ended
  wordPair: {                    // 本局的词对
    civilian: String,
    spy: String,
    difficulty: 1 | 2 | 3,
  },
  currentSpeakerIndex: Number,   // 当前描述者的索引
  descriptions: [{               // 所有描述记录
    playerId: String,
    content: String,
    round: Number,
  }],
  votes: {                       // 投票记录: { 投票者ID: 被投者ID }
    [voterId]: targetId,
  },
  guessResult: {                 // 猜词结果
    playerId: String | null,
    guessedWord: String | null,
    isCorrect: Boolean | null,
  },
  eliminated: [{                 // 出局记录
    playerId: String,
    round: Number,
    byVote: Boolean,             // true=被投出, false=猜错暴露
  }],
  winner: String | null,         // civilian | spy | white
  isSpecialWin: Boolean,         // 是否特殊胜利
}
```

### 特殊情况处理

| 场景 | 处理方式 |
|------|---------|
| 平票 | 无人出局，直接进入猜词阶段 |
| 只剩最后1个卧底且被投出 | 平民胜利 |
| 白板每轮投票后都可猜词 | 但全局限一次猜对机会 |
| 卧底全部出局但平民还有白板 | 白板视为卧底方 |
| 玩家中途断线 | 标记为离线，本轮存活不投票，下一局移出 |

## 七、词库设计

### 词对格式

```javascript
{
  civilian: "平民词",
  spy: "卧底词",
  difficulty: 1 | 2 | 3,       // 难度
  category: "生活/娱乐/科技/食物/人物",
  hint: "可选提示（用于白板参考提示模式）"
}
```

### 难度定义

| 难度 | 区分度 | 例子 | 适合 |
|------|--------|------|------|
| ⭐ 简单 | 明显不同 | 苹果 vs 梨 | 新手局 |
| ⭐⭐ 中等 | 比较接近 | 口红 vs 唇膏 | 普通局 |
| ⭐⭐⭐ 困难 | 非常接近 | 高铁 vs 动车 | 高手局 |

### 词库示例（50+ 组）

| 平民词 | 卧底词 | 难度 | 分类 |
|--------|--------|------|------|
| 苹果 | 梨 | ⭐ | 水果 |
| 薯条 | 薯片 | ⭐ | 食物 |
| 西瓜 | 冬瓜 | ⭐ | 水果 |
| 牛奶 | 豆浆 | ⭐ | 饮品 |
| 钢笔 | 铅笔 | ⭐ | 文具 |
| 口红 | 唇膏 | ⭐⭐ | 美妆 |
| 电梯 | 扶梯 | ⭐⭐ | 设施 |
| 警察 | 保安 | ⭐⭐ | 职业 |
| 眼镜 | 墨镜 | ⭐⭐ | 配饰 |

## 八、部署方案

### 方式一：一键部署到 Railway（推荐）

1. 代码推送到 GitHub
2. 在 Railway 选择这个仓库
3. 自动部署，获得 `xxx.railway.app` 域名
4. 分享链接给朋友就完事
5. Railway 免费套餐：每月 500 小时运行时长，足够小范围使用

### 方式二：本地运行 + 内网穿透

1. 本地 `node server.js`
2. 用 ngrok 映射到公网 `ngrok http 3000`
3. 分享 ngrok 给的临时链接
4. 适合快速测试，临时玩几局

### 方式三：Vercel + WebSocket 服务

- 前端部署到 Vercel
- WebSocket 服务部署到 Render / Fly.io
- HTTP 请求走 Vercel Serverless，WS 连独立服务

## 十、扩展性考虑（将来可加）

- **语音描述**：长按录音，实时播放给其他人（WebRTC 集成）
